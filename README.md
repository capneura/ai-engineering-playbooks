## Purpose

This repository tracks the development of the manuscript for **Engineering With AI Without Losing Control**.

The book documents practical engineering methods learned during long-running AI-assisted software development work.

It is not a book about prompts.

It is not a book about ChatGPT.

It is not a collection of AI tricks.

It is a field manual about preserving engineering control while using AI as part of real project development.

## Core Argument

AI does not remove engineering discipline.

It punishes the absence of it faster.

The more capable the assistant becomes, the more important it is to preserve objectives, evidence, boundaries, decisions, and verification.

## Author Position

The author is presented honestly as an **Independent Systems Practitioner** documenting practical methods learned while building long-running AI-assisted engineering projects.

No exaggerated credentials.

No claims of being an AI expert.

No borrowed authority.

The work stands on demonstrated experience, documented process, and extracted engineering principles.

## Editorial Rules

### Rule 1: Evidence First

Nothing enters the manual unless it can be traced back to real engineering experience.

### Rule 2: Generalize the Lesson, Not the Project

The original project is not named in the manuscript.

Only the engineering principle survives.

### Rule 3: Every Recommendation Must Answer Four Questions

Each major recommendation should answer:

1. What happened?
2. What was learned?
3. What engineering rule emerged?
4. How can another builder apply it?

### Rule 4: Do Not Invent Methodology

The manuscript does not invent a process and pretend it came from experience.

It extracts methodology from actual engineering work.

## Working Method

This repository is treated as an engineering repository whose product happens to be a book.

The writing process follows the same discipline promoted by the manuscript:

1. Extract evidence.
2. Identify the principle.
3. Draft the chapter.
4. Review the argument.
5. Revise the manuscript.
6. Freeze the milestone.
7. Commit the result.
